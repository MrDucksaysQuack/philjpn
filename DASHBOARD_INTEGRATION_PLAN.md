**목표**: 모든 구현된 기능이 대시보드에서 직접 접근 가능하도록 통합

## 🔍 현재 상태 분석

### 현재 문제점

1. **메인 페이지 (`/page.tsx`)**
   - ❌ 로그인 여부와 관계없이 동일한 Hero 섹션만 표시
   - ❌ 사용자의 학습 상황을 한눈에 볼 수 없음
   - ❌ 새로 구현된 기능들에 대한 접근이 Header 링크에만 의존

2. **기존 페이지들**
   - ✅ `/analysis` - 자기 분석 대시보드 (구현 완료)
   - ✅ `/exams/recommended` - 추천 시험 (구현 완료)
   - ✅ `/exams` - 시험 목록 (추천 링크만 있음)
   - ❌ 통합 대시보드 없음

3. **Admin 대시보드**
   - ✅ `/admin` - 잘 구성된 관리자 대시보드
   - ✅ 빠른 링크로 모든 기능 접근 가능

### 구현된 기능 목록 (COGNITIVE_LEARNING_IMPROVEMENTS.md 기준)

**사용자 기능**:
1. ✅ 자기 분석 대시보드 (`/analysis`)

6. ✅ 학습 사이클

**Admin 기능**:
1. ✅ 시험 템플릿 관리
2. ✅ 문제 풀 관리
3. ✅ 분석 도구

## 🎯 개선 방안

### ✅ 선택된 방식: 별도 사용자 대시보드 페이지 생성

**접근 방식**:
- ✅ 메인 페이지 (`/page.tsx`)는 **그대로 유지** (Hero 섹션)
- ✅ 새로운 사용자 대시보드 페이지 (`/dashboard`) 생성
- ✅ 로그인 사용자는 메인 페이지에서 대시보드 링크로 접근
- ✅ Header에 "대시보드" 메뉴 추가

**접근 경로**:
```
메인 페이지 (/) → Hero 섹션
  ├─ 로그인 사용자: "나의 대시보드" 버튼 → /dashboard
  └─ 비로그인 사용자: "시험 시작하기" → /exams

Header 메뉴 → "대시보드" → /dashboard
```

## 📋 구현 계획

### Phase 1: 사용자 대시보드 페이지 생성 ✅

**구현 내용**:

1. **새 대시보드 페이지 생성**
   ```typescript
   // app/dashboard/page.tsx
   - 로그인 체크 (미로그인 시 /login으로 리다이렉트)
   - 모든 위젯 컴포넌트 통합
   ```

2. **대시보드 섹션 구성** (인지 패턴 원리 적용)
   - **빠른 통계 카드** (4개) - 가장 중요한 정보 우선
     - 오늘 응시한 시험 수
     - 이번 주 목표 진행률
     - 최근 평균 점수
     - 학습 중인 단어 수
   
   - **목표 진행 상황 위젯** - 의미적 계층화 (최우선 표시)
     - 가장 중요한 목표 1개 대형 표시
     - 진행률 바 (감정적 색상)
     - 남은 기간, 다음 마일스톤
     - "목표 설정하기" 버튼 (목표 없을 때)
     - "자세히 보기" 링크
   
   - **추천 시험 위젯** - 맥락 기반 반응성
     - 상위 3개 추천 시험 카드
     - 적합도 점수, 추천 이유
     - "모든 추천 시험 보기" 링크
   
   - **최근 학습 활동 위젯** - 내러티브 구조
     - 최근 응시한 시험 3개
     - 점수, 개선 추이 (↑↓)
     - "전체 결과 보기" 링크
   
   - **학습 패턴 인사이트 위젯** - 인지 부하 최소화
     - 최적 학습 시간대 (간단한 텍스트)
     - 주요 약점 영역 (상위 3개, 태그만)
     - 효율성 지표 요약 (주당 향상률)
     - "자세한 분석 보기" 링크
   
   - **내 단어장 요약 위젯** - 개인화 정보 통합
     - 최근 추가한 단어 5개
     - 학습 중인 단어 수
     - "단어장으로 가기" 링크
   
   - **빠른 액션 섹션** - 명확한 다음 단계 제시
     - "시험 시작하기" → /exams
     - "자기 분석 보기" → /analysis
     - "통계 확인하기" → /statistics

3. **API 호출 최적화**
   - React Query 병렬 로딩
   - 필요한 데이터만 선택적 로딩
   - 로딩 상태 스켈레톤/스피너
   - 에러 처리 및 재시도

4. **메인 페이지 개선**
   - 로그인 사용자에게 "나의 대시보드" 버튼 추가
   - Hero 섹션에 대시보드 링크 배치

5. **Header 네비게이션 개선**
   - "대시보드" 메뉴 항목 추가 (로그인 사용자에게만)
   - 순서: 시험 목록 | 내 결과 | 단어장 | 통계 | **대시보드** | 자기 분석

## 🎨 인지 패턴 원리 적용

### 1. 의미적 계층화 (Semantic Hierarchy)
- **목표 진행 위젯**을 최상단에 배치 (가장 중요한 정보)
- 빠른 통계 카드를 상단에 배치하여 한눈에 파악 가능
- 각 위젯 크기와 색상으로 중요도 표현

### 2. 맥락 기반 반응성 (Contextual Reactivity)
- 사용자의 현재 상태에 맞는 추천 시험 제시
- 최근 활동 기반으로 다음 액션 제안
- "지금 해야 할 것" 명확히 표시

### 3. 내러티브 구조 (Narrative Flow)
- 학습 여정을 시간 순서로 표현 (최근 활동 → 추천 → 목표)
- "어디서 왔는가 → 지금 어디인가 → 어디로 가야 하는가" 흐름
- 빠른 액션 섹션으로 다음 단계 명확히 제시

### 4. 감정적 피드백 (Affective Feedback)
- 목표 진행률 색상으로 감정 전달 (빨강→노랑→초록)
- 격려 메시지 ("🎯 거의 다 왔어요!")
- 성취감 강조 (진행률, 개선 추이)

### 5. 인지 부하 최소화 (Cognitive Load Reduction)
- 각 위젯은 핵심 정보만 표시
- 상세 정보는 링크로 분리
- 빠른 통계는 숫자와 아이콘만으로 직관적 표현
