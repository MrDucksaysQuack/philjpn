## 1️⃣ 인지 부하 최소화 (Cognitive Load Reduction)

### 현재 적용도: 75% 🟡

#### ✅ 잘 구현된 부분

1. **색상 의미 체계**
   - ✅ 파란색 = 긍정/진행: `bg-blue-600` (시험 시작, 로그인)
   - ✅ 빨간색 = 위험/중요: `bg-red-600` (시험 제출)
   - ✅ 회색 = 중립/비활성: `disabled:opacity-50`

2. **버튼 구조**
   - ✅ Primary Action이 항상 명확 (그라데이션, 큰 크기)
   - ✅ Secondary Action은 테두리 스타일
   ```tsx
   // 예시: login/page.tsx
   Primary: bg-gradient-to-r from-blue-600 to-purple-600
   Secondary: bg-transparent border-2 border-white
   ```

3. **아이콘 사용**
   - ✅ 의미 명확한 SVG 아이콘 (체크, X, 화살표)
   - ✅ Toast 컴포넌트의 타입별 아이콘 구분

#### ⚠️ 개선 필요 부분

1. **맥락 기반 버튼 위치**
   - ❌ 현재: 모든 페이지에서 버튼 위치가 일관적이지 않음
   - ✅ 개선: 주요 액션은 항상 우측 하단 또는 상단 우측 고정
   
   **예시 문제**:
   ```tsx
   // exams/[id]/take/page.tsx - 버튼이 중앙에 흩어져 있음
   <div className="flex justify-between">
     <button>이전</button>
     <button>다음</button>
   </div>
   ```

2. **복잡한 메뉴 구조**
   - ❌ Header 메뉴가 텍스트 중심으로만 구성
   - ✅ 개선: 자주 사용하는 기능은 아이콘 + 텍스트로 직관화

3. **폼 필드 그룹화**
   - ⚠️ 일부 폼이 긴 스크롤 구조 (회원가입, 시험 생성)
   - ✅ 개선: 단계별 진행 바 또는 섹션 구분

### 개선 권장사항

1. **Floating Action Button (FAB) 패턴 도입**
   - 주요 액션 버튼을 화면 우측 하단 고정
   - 예: "시험 시작하기", "결과 저장하기"

2. **Breadcrumb 네비게이션 강화**
   - 현재 위치 명확화
   - 이전 단계로 빠른 이동

3. **프로그레스 인디케이터**
   - 시험 응시 중 진행률 표시
   - 폼 입력 중 단계 표시

## 2️⃣ 의미적 계층화 (Semantic Hierarchy)

### 현재 적용도: 85% ✅

#### ✅ 잘 구현된 부분

1. **시각적 계층**
   - ✅ H1 > H2 > H3 텍스트 크기 체계 명확
   - ✅ 그라데이션 배경으로 섹션 구분
   ```tsx
   // analysis/page.tsx - 명확한 시각적 계층
   <h1 className="text-4xl sm:text-5xl font-extrabold">자기 분석 대시보드</h1>
   <h2 className="text-2xl font-bold text-gray-900">시간대별 학습 패턴</h2>
   ```

2. **색상으로 중요도 표시**
   - ✅ 그라데이션 바 (`w-1 h-8 bg-gradient-to-b`)로 섹션 시작 표시
   - ✅ 카드 배경색으로 정보 그룹화
   
3. **공간 배치**
   - ✅ 상단 = 중요 정보 (제목, 통계)
   - ✅ 중앙 = 주요 콘텐츠
   - ✅ 하단 = 보조 정보

4. **Typography 체계**
   - ✅ `font-extrabold` = 가장 중요한 정보
   - ✅ `font-semibold` = 부제목/라벨
   - ✅ `text-gray-500` = 보조 텍스트

#### ⚠️ 개선 필요 부분

1. **Primary Action 시각적 강조**
   - ⚠️ 일부 페이지에서 Primary/Secondary 구분이 모호
   - ✅ 개선: Shadow, 크기 차이로 더 명확히 구분

2. **정보 밀도**
   - ⚠️ 일부 대시보드가 정보 과다 (analysis 페이지)
   - ✅ 개선: 접을 수 있는 섹션 (Accordion) 도입

### 개선 권장사항

1. **Primary Action 강조 패턴**
   ```tsx
   Primary: shadow-xl + transform hover:scale-105
   Secondary: shadow-md
   Tertiary: shadow-sm 또는 border only
   ```

2. **Collapsible 섹션**
   - 긴 분석 결과는 기본 접힘 상태
   - "자세히 보기" 버튼으로 확장

## 3️⃣ 맥락 기반 반응성 (Contextual Reactivity)

### 현재 적용도: 60% 🟡 ⚠️

#### ✅ 잘 구현된 부분

1. **로딩 상태 표시**
   - ✅ `LoadingSpinner`로 명확한 로딩 피드백
   - ✅ 메시지 포함: "학습 패턴을 분석하는 중..."

2. **에러 메시지 기본 구조**
   - ✅ Toast 시스템으로 전역 에러 처리
   - ✅ 타입별 색상 구분 (success/error/info/warning)

#### ❌ 개선 필요 부분 (중요!)

1. **에러 메시지가 절차적, 맥락 부족**
   ```tsx
   // 현재 (절차적)
   "시험 시작에 실패했습니다."
   "오류가 발생했습니다."
   "결과를 불러오는데 실패했습니다."
   
   // 개선 (맥락 기반)
   "네트워크 연결이 불안정합니다. 다시 시도할까요?"
   "라이선스 키가 만료되었습니다. 새 키를 발급받으세요."
   "서버가 일시적으로 응답하지 않습니다. 30초 후 자동 재시도됩니다."
   ```

2. **성공 메시지도 맥락 부족**
   ```tsx
   // 현재
   toast.success("성공")
   
   // 개선 필요
   toast.success("시험이 제출되었습니다. 결과를 확인하세요!")
   toast.success("단어 5개가 단어장에 추가되었습니다")
   ```

3. **상황별 제안 부족**
   - ❌ 에러 발생 시 다음 액션 제안 없음
   - ❌ 네트워크 오류 시 재시도 버튼 없음
   - ❌ 빈 상태(Empty State) 메시지 단순

### 개선 권장사항 (우선순위: 높음)

1. **맥락 기반 에러 메시지 템플릿**
   ```tsx
   interface ErrorContext {
     type: 'network' | 'validation' | 'permission' | 'server';
     message: string;
     action?: {
       label: string;
       onClick: () => void;
     };
     retry?: boolean;
   }
   ```

2. **에러 유형별 대응 제안**
   - 네트워크 오류 → "재시도" 버튼 제공
   - 권한 오류 → "로그인 페이지로 이동" 버튼
   - 검증 오류 → 문제 필드로 스크롤 + 포커스

3. **상황 인식 메시지**
   ```tsx
   // 시험 제출 후
   "🎯 시험이 제출되었습니다. 결과 분석이 완료되면 알림을 드리겠습니다."
   
   // 단어 추가 시
   "✨ 3개의 단어가 추가되었습니다. 지금 복습하시겠어요?"
   ```

## 4️⃣ 내러티브 구조 (Narrative Flow)

### 현재 적용도: 70% 🟡

#### ✅ 잘 구현된 부분

1. **시험 응시 플로우**
   ```
   시험 목록 → 시험 상세 → 시험 시작 → 시험 응시 → 결과 확인
   ```
   - ✅ 각 단계 간 자연스러운 전환
   - ✅ 명확한 다음 단계 표시

2. **학습 분석 플로우**
   ```
   시험 응시 → 결과 분석 → 약점 파악 → 목표 설정 → 추천 시험
   ```
   - ✅ 탭 기반 네비게이션으로 단계 구분
   - ✅ 결과 페이지에서 다음 액션 제안

#### ⚠️ 개선 필요 부분

1. **프로그레스 표시 부족**
   - ❌ 시험 응시 중 진행률 표시 없음
   - ❌ 폼 제출 중 단계 표시 없음
   
2. **스토리 중단**
   - ⚠️ 결과 페이지에서 "다음 단계" 링크가 텍스트만
   - ✅ 개선: 큰 버튼으로 "약점 분석 보기" 제안

3. **복귀 경로 불명확**
   - ⚠️ 일부 페이지에서 뒤로가기 경로 모호
   - ✅ 개선: Breadcrumb 또는 명확한 "돌아가기" 버튼

### 개선 권장사항

1. **진행률 표시**
   ```tsx
   // 시험 응시 중
   <ProgressBar 
     current={answeredQuestions} 
     total={totalQuestions}
     message="문제 15/50 완료"
   />
   ```

2. **다음 단계 강조**
   ```tsx
   // results/[id]/page.tsx
   <div className="mt-8 p-6 bg-gradient-to-r from-blue-50 to-purple-50 rounded-xl">
     <h3 className="text-lg font-semibold mb-2">다음 단계</h3>
     <Link href="/analysis" className="inline-block">
       <button className="bg-blue-600 text-white px-6 py-3 rounded-lg">
         내 학습 패턴 분석하기 →
       </button>
     </Link>
   </div>
   ```

3. **스토리텔링 요소 추가**
   - 완료 시: "축하합니다! 🎉 이번 시험에서 X점을 달성했습니다."
   - 개선 시: "이전 시험 대비 +5점 향상! 계속 화이팅! 💪"

## 5️⃣ 감정적 피드백 (Affective Feedback)

### 현재 적용도: 50% 🔴 ⚠️

#### ✅ 잘 구현된 부분

1. **시각적 보상**
   - ✅ 그라데이션 효과로 성공 감각 제공
   - ✅ 아이콘 사용 (체크마크, 별 등)

2. **애니메이션**
   - ✅ `animate-fade-in`, `hover:scale-105` 등 부드러운 전환
   - ✅ Toast 슬라이드 인 애니메이션

#### ❌ 개선 필요 부분 (중요!)

1. **성공 메시지가 논리적, 감정 부족**
   ```tsx
   // 현재 (논리적)
   "✅ 성공적으로 처리되었습니다"
   "저장되었습니다"
   
   // 개선 (감정적)
   "🎉 완료! 멋진 선택이에요!"
   "✨ 단어가 추가되었습니다. 지금 복습해볼까요?"
   "💪 잘했어요! 이번 시험에서 5점 향상했어요!"
   ```

2. **목표 달성 보상 부족**
   - ❌ 목표 달성 시 특별한 피드백 없음
   - ✅ 개선: 축하 메시지 + 배지 또는 이모지

3. **진행 상황 감정적 표현 부족**
   - ❌ 단순 숫자 표시 (예: "75점")
   - ✅ 개선: "75점! 아직 5점만 더하면 목표 달성이에요! 🎯"

4. **에러 메시지가 차갑고 기계적**
   - ❌ "오류가 발생했습니다"
   - ✅ 개선: "앗, 문제가 생겼네요. 걱정 마세요, 다시 시도해볼까요?"

### 개선 권장사항 (우선순위: 높음)

1. **감정적 언어 패턴 도입**
   ```tsx
   const messages = {
     success: {
       examSubmitted: "🎉 시험이 제출되었습니다! 결과 확인하기",
       wordAdded: "✨ 단어가 추가되었습니다. 지금 복습하시겠어요?",
       goalAchieved: "🏆 목표 달성! 멋져요! 다음 목표를 설정해볼까요?",
     },
     error: {
       network: "😓 연결이 불안정해요. 잠시 후 다시 시도해볼까요?",
       validation: "⚠️ 입력 내용을 확인해주세요",
       permission: "🔒 로그인이 필요해요",
     }
   };
   ```

2. **목표 달성 축하 UI**
   ```tsx
   <CelebrationModal 
     title="🎉 목표 달성!"
     message="900점 목표를 달성하셨어요! 정말 멋져요!"
     emoji="🎊"
   />
   ```

3. **진행 상황 감정 표현**
   ```tsx
   // 목표 진행 중
   <div className="text-center">
     <div className="text-4xl font-bold text-blue-600">
       {currentScore}점
     </div>
     <p className="text-lg text-gray-600 mt-2">
       {currentScore >= targetScore * 0.9 
         ? "🎯 거의 다 왔어요! 화이팅!" 
         : "💪 좋아요! 계속 달려봐요!"}
     </p>
   </div>
   ```

4. **개인화된 격려 메시지**
   ```tsx
   const encouragementMessages = [
     "🔥 오늘도 열심히 하셨네요!",
     "⭐ 이번 주 3번째 시험이에요. 대단해요!",
     "🚀 계속 향상되고 있어요!",
   ];
   ```
